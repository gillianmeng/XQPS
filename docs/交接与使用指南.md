# XQPS 使用与交接指南

面向**接手运维/开发/管理员**的说明：部署、权限、配置、风险与日常操作。

---

## 1. 系统是什么

- **XQPS**：绩效管理系统，**Streamlit** 单进程 Web 应用，数据来自**飞书多维表**（Bitable），通过飞书开放平台 **OAuth** 登录。
- **主入口**：`new_app.py`（约 7000+ 行，业务逻辑集中在此）。
- **角色**：员工、管理者、一级部门负责人、分管高管、**系统管理员**、**HRBP / HRBP Lead** 等，由飞书多维表字段（如 `后台角色`、`姓名`、部门与负责人关系等）与登录逻辑共同判定。

## 3. 环境与密钥（接手必查）

### 3.1 必填（飞书应用与多维表）

在 **环境变量** 或 **`.streamlit/secrets.toml`** 中配置（代码中 `missing` 检查会 `st.stop()`）：

| 变量/键 | 含义 |
|---------|------|
| `FEISHU_APP_ID` | 飞书应用 ID |
| `FEISHU_APP_SECRET` | 飞书应用 Secret |
| `FEISHU_APP_TOKEN` | 多维表 App Token |
| `FEISHU_TABLE_ID` | 多维表 Table ID |
| `REDIRECT_URI` | OAuth 回调 URL，**必须与飞书开放平台应用里配置的完全一致**（生产为公网 HTTPS） |

### 3.2 常用可选

| 变量 | 说明 |
|------|------|
| `APP_ENV` | `production` / `staging` 等，影响是否走演示逻辑等 |
| `ENABLE_DEMO_LOGIN` | `true` 时允许演示入口（**生产务必 `false`**） |
| `ENABLE_DEV_TOOLS` | 开发辅助，生产保持关闭 |
| `XQPS_CONFIG_DIR` | **管理员配置文件目录**。不设时，代码会尝试项目目录可写路径或 `~/xqps_config`。Docker/只读盘**必须**指向可挂载、可写目录，否则后台无法保存 |
| `XQPS_USE_FROZEN_ANNOUNCEMENTS` | `1`/`true`：公告强制使用仓库内 `frozen_announcements.json`（见第 5 节） |

### 3.3 飞书侧权限（非代码内）

接手时需向原负责人确认并迁移：

- 飞书**应用负责人**、**凭证轮换**责任；
- 开放平台里 **重定向 URL**、**权限范围**（通讯录、多维表等）是否仍满足；
- 多维表 **字段变更**（列名、选项）会直接影响本应用解析逻辑，属**业务风险点**。

## 4. 运行与部署

```bash
pip install -r requirements.txt
streamlit run new_app.py --server.port 8501
```

生产建议：`APP_ENV=production`、`ENABLE_DEMO_LOGIN=false`。

**脚本**：`bin/start_server.sh` 会设置默认 `XQPS_CONFIG_DIR=/data/xqps_config` 并尝试创建目录（便于后台配置可写）。详见 `docs/飞书多维表格迁移说明.md`。

**进程模型**：Streamlit **默认单 worker**；多副本部署时，每台实例各自读写本地/挂载盘上的 `admin_config.json`，**不存在应用内分布式锁**（见第 7 节）。

## 5. 管理员配置与公告

### 5.1 `admin_config.json`

- **路径**由 `_resolve_admin_config_path()` 决定（环境变量 > 可写项目目录 > `~/xqps_config`）。
- **修改方式**：管理员登录后在侧边栏 **「📋 后台配置」** 中操作；代码注释亦要求**不要**依赖手工改文件作为常规流程。
- **内容**：展示周期 `display_cycle`、按周期的 `cycle_configs`（公告 `announcements`、制度链接 `doc_links`、模块开关等）、结果是否开放查看等。

### 5.2 `frozen_announcements.json`（内置快照）

- 与 `new_app.py` **同目录**，可随版本发布。
- **作用**：当 **不存在** `admin_config.json`（或需要强制）时，各模块「公告」回退到该文件中的文案（逻辑见 `_read_admin_announcement`）。
- **强制使用内置**：设置 `XQPS_USE_FROZEN_ANNOUNCEMENTS=1`（例如后台目录暂时不可写、又需统一展示发版文案）。

发版前若线上公告已更新：可从可写环境导出 `admin_config` 中对应周期的 `announcements`，合并进 `frozen_announcements.json`（按周期键与 `_default`）。

## 7. 技术风险与并发（接手必读）

### 7.1 Streamlit 执行模型

- 每次交互会**从上到下重新执行**脚本；状态依赖 **`st.session_state`**（按浏览器会话隔离）。
- **无**传统多线程业务锁；同一用户快速点击可能导致重复提交，敏感写操作依赖飞书 API 幂等/校验。

### 7.2 `admin_config.json` 写入

- **读-改-写**整文件；两名管理员同时保存时，**后写可能覆盖先写**（last-write-wins）。
- 缓解：单管理员改配置；或运维上限制并发、缩短操作窗口；重大变更前备份文件。

### 7.3 缓存

- `@st.cache_data` 等用于飞书拉表等；租户 Token、表数据缓存 TTL 见代码。飞书侧数据变更后，用户可能需等待 TTL 或刷新会话才看到最新（具体以函数装饰器为准）。

### 7.4 HRBP 与「高管」数据

- HRBP 视图在组装 `report_records` 后会过滤「特殊判断 = 高管」人员；与管理员全量视图口径不同，**交接时勿混淆报表范围**。

### 7.5 周期字符串

- `2026上半年` 与 `2026年上半年` 在代码中按 **等价周期** 处理（`CYCLE_EQUIVALENTS`）；新增周期时保持与飞书字段、管理员配置一致。
