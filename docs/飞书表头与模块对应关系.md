# 飞书多维表字段与模块对应关系

> 基于 `new_app.py` 梳理。以**表头（列名）**为主键，便于建表、列核对与权限排查；文末为别名、工作目标命名规则、写入汇总与建表顺序。

---

## 〇、说明（与代码一致）

| 项 | 说明 |
|----|------|
| **公告「📢」** | 员工自评 / 上级评分 / 一级部门负责人调整 / 分管高管调整 顶部的公告文案来自 **`admin_config.json`**，无配置时可回退 **`frozen_announcements.json`**。**不是**从飞书多维表读取；当前应用**未使用**表中「自评提交截止日期」「上级评价提交截止日期」等列。若表中保留此类列，仅为业务留档或他处使用，与页面公告无关。 |
| **`后台角色`** | 用于**系统管理员**、**HRBP / HRBP Lead** 等登录后权限判定。 |
| **`特殊判断`** | 值为「高管」时，在 HRBP 等视图中**不参与**统计与展示（见代码 `_is_executive`）；管理员视图仍可按产品设计展示。 |

## 四、工作目标字段命名规则

- `工作目标{i}及总结`：i = 1 ~ 5  
- `工作目标{i}权重`：i = 1 ~ 5  
- `工作目标{i}自评得分`：i = 1 ~ 5  
- `工作目标{i}上级评分`（分项）：代码中**未使用**；仅使用 **`工作目标上级评分`**（单列整体存储上级对工作目标的评分）
