> 基于 `new_app.py` 代码梳理。涵盖员工自评、上级评分、一级部门负责人调整、分管高管调整四类确认提交。

---

## 一、概览

| 场景 | 按钮文案 | 可提交条件 | 禁用条件 | 提交后效果 |
|------|----------|------------|----------|------------|
| 员工自评 | 确认提交 | 权重+总结+得分全部合规 | 未填完 / 已提交 | 自评是否提交=是，表单锁定 |
| 上级评分 | 确认提交 | 全部直属有等级 且 至少一人未提交 | 未填完 / 全部已提交 | 上级评价是否完成=是 |
| 一级部门负责人调整 | 确认本次调整 | 有待调整 且 全部待调整项有等级 | 无待调整 / 未填完 | 一级部门调整完毕=是 |
| 分管高管调整 | 确认本次调整 | 有待调整 且 全部待调整项有等级 | 无待调整 / 未填完 | 分管高管调整完毕=是 |

## 二、员工自评 - 确认提交

### 2.1 位置

- Tab：📝 员工自评
- Marker：`.self-eval-submit-marker`

### 2.2 可提交条件 `step1_can_submit`

```python
step1_can_submit = weight_valid and summaries_valid and scores_valid
```

| 校验项 | 规则 |
|--------|------|
| **weight_valid** | 工作目标权重之和 = target_weight（员工 80%，管理者 60%） |
| **summaries_valid** | 所有总结非空、长度 100–5000 字 |
| **scores_valid** | 所有自评得分非 0 |

**总结校验：**
- 工作目标 1–5：每项有总结，100–5000 字
- 通用能力：有总结，100–5000 字
- 领导力（仅管理者）：有总结，100–5000 字

**得分校验：**
- 工作目标 1–5：每项自评得分 ≠ 0
- 通用能力自评得分 ≠ 0
- 领导力自评得分（仅管理者）≠ 0

### 2.3 禁用条件

- `disabled=not step1_can_submit`
- 已提交（`自评是否提交=是`）时，表单整体 disabled，按钮不显示或不可点

### 2.4 提交动作

- 写入：工作目标、通用能力、领导力（管理者）、自评得分、自评等级、**自评是否提交=是**
- 提交后表单锁定，不可再修改

### 2.5 按钮样式

- 未填完：灰色 `#5a5a5a`
- 填完且合规：红色 `#d32f2f`

## 三、上级评分 - 确认提交

### 3.1 位置

- Tab：👥 上级评分
- Marker：`.mgr-submit-marker`
- Key：`btn_mgr_confirm_all`

### 3.2 可提交条件 `mgr_can_confirm_all`

```python
# 1. 全部直属须有考核等级（未提交的须已暂存）
mgr_can_confirm_all = True
for _s in my_direct_subs:
    _done = 上级评价是否完成 == "是"
    _grade = 考核结果
    if not _done and _grade not in GRADE_OPTIONS:
        mgr_can_confirm_all = False
        break

# 2. 全部已提交则不可再提交
if my_direct_subs and all(上级评价是否完成 == "是" for _s in my_direct_subs):
    mgr_can_confirm_all = False
```

| 条件 | 说明 |
|------|------|
| 全部有等级 | 每个直属的「考核结果」在 S/A/B+/B/B-/C 中，或已提交 |
| 至少一人未提交 | 至少一个直属的「上级评价是否完成」≠ 是 |

### 3.3 禁用条件

- `disabled=not mgr_can_confirm_all`
- 存在直属未填考核等级 → 禁用
- 全部直属已提交 → 禁用（不可再提交）

### 3.4 提交动作

- 遍历 `my_direct_subs`
- 跳过：`上级评价是否完成=是` 或 `考核结果` 不在等级内
- 写入：`上级评价是否完成=是`
- 若本人为一级部门负责人且下属为本人负责：同时写入 `一级部门调整考核结果`、`一级部门调整完毕=是`

### 3.5 按钮样式

- 未全部暂存：灰色
- 全部暂存且有待提交：红色

## 四、一级部门负责人调整 - 确认本次调整

### 4.1 位置

- Tab：📌 一级部门负责人调整
- Marker：`.dept-confirm-marker`
- Key：`btn_dept_confirm_all`

### 4.2 可提交条件

```python
dept_has_any_pending = False
dept_can_confirm_all = True
for rec in dept_head_records:
    mgr_g = 考核结果
    done_f = 一级部门调整完毕 == "是"
    if mgr_g not in GRADE_OPTIONS or done_f:
        continue
    dept_has_any_pending = True
    sel_grade = st.session_state.get(f"dept_adj_grade_{r_id}", def_grade)
    if sel_grade not in GRADE_OPTIONS:
        dept_can_confirm_all = False
        break
dept_btn_disabled = not dept_has_any_pending or not dept_can_confirm_all
```

| 条件 | 说明 |
|------|------|
| **dept_has_any_pending** | 至少一人：考核结果有效 且 一级部门调整完毕≠是 |
| **dept_can_confirm_all** | 所有待调整人的选中等级在 S/A/B+/B/B-/C 中 |

### 4.3 禁用条件

- `disabled=dept_btn_disabled`
- 无待调整人 → 禁用
- 有待调整人但存在未选等级 → 禁用

### 4.4 提交动作

- 遍历 `dept_head_records`
- 跳过：考核结果无效 或 一级部门调整完毕=是
- 写入：`一级部门调整考核结果`、`一级部门调整完毕=是`

### 4.5 按钮样式

- 未全部填写：灰色
- 全部填写：红色

## 五、分管高管调整 - 确认本次调整

```python
vp_has_any_pending = False
vp_can_confirm_all = True
for rec in vp_records:
    dept_done = 一级部门调整完毕 == "是"
    vp_done = 分管高管调整完毕 == "是"
    if not dept_done or vp_done:
        continue
    vp_has_any_pending = True
    sel_grade = st.session_state.get(f"vp_adj_grade_{r_id}", def_grade)
    if sel_grade not in GRADE_OPTIONS:

```

| 条件 | 说明 |
|------|------|
| **vp_has_any_pending** | 至少一人：一级部门调整完毕=是 且 分管高管调整完毕≠是 |
| **vp_can_confirm_all** | 所有待调整人的选中等级在 S/A/B+/B/B-/C 中 |

### 5.3 禁用条件

- `disabled=vp_btn_disabled`
- 无待调整人（一级部门未完成或分管高管已全部完成）→ 禁用

### 5.4 提交动作

- 遍历 `vp_records`
- 跳过：一级部门调整完毕≠是 或 分管高管调整完毕=是
- 写入：`分管高管调整考核结果`、`分管高管调整完毕=是`

### 5.5 按钮样式

- 与一级部门负责人调整一致：未填完灰色，全部填写红色

## 六、统一规则

### 6.1 提交后不可修改

- 员工自评：`自评是否提交=是` 后表单锁定
- 上级评分：`上级评价是否完成=是` 后不可再提交
- 一级部门负责人：`一级部门调整完毕=是` 后不可再改
- 分管高管：`分管高管调整完毕=是` 后不可再改

### 6.2 按钮配色

| 状态 | 背景色 | 说明 |
|------|--------|------|
| 禁用 | `#5a5a5a` | 灰色 |
| 可点击 | `#d32f2f` | 红色 |
| 悬停 | `#b71c1c` | 深红 |

### 6.3 提示文案

- 员工自评：「点击「确认提交」即意味着本次自评结束，不可再修改。」
- 上级评分：「点击「确认提交」即意味着对全部下属的本次评估结束，不可再修改。」
- 一级部门/分管高管：「默认为前序调整结果。全部调整完毕请点击「确认本次调整」按钮。提交后不可再修改。」
