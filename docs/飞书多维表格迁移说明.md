# 飞书多维表格 / XQPS 系统迁移说明

> 本文档说明如何将 XQPS 绩效管理系统从当前部署环境迁移到管理员电脑或新地址，确保迁移顺利、不丢失配置。

---

## 一、迁移概述

| 项目 | 说明 |
|------|------|
| **数据位置** | 绩效数据存储在**飞书多维表格（云端）**，迁移应用服务器不会丢失数据 |
| **需要迁移** | 应用代码、配置文件、后台配置（公告、展示周期等） |
| **涉及角色** | 迁移方（你）、接收方（管理员）、飞书应用管理员（如需改配置） |

---

## 二、迁移方（你）需要做的工作

### 2.1 提供代码与配置

1. **代码仓库**
   - 从 GitLab 拉取最新 `prod` 分支，或提供打包代码（zip）
   - 仓库地址：`http://git.snowballfinance.com/hr/xqps.git`

2. **密钥配置（必须、保密）**
   - 将以下配置整理成一份清单，**通过安全方式**（如企业微信/面对面）提供给管理员，**勿通过邮件明文发送**
   - 配置项示例（具体值由你提供）：

   | 配置项 | 说明 | 获取位置 |
   |--------|------|----------|
   | FEISHU_APP_ID | 飞书应用 ID | 飞书开放平台 → 应用概况 |
   | FEISHU_APP_SECRET | 飞书应用密钥 | 飞书开放平台 → 凭证与基础信息 |
   | FEISHU_APP_TOKEN | 多维表格 App Token | 多维表格 → 分享 → 复制链接中的 token |
   | FEISHU_TABLE_ID | 多维表格 Table ID | 同上，链接中的 table 参数 |
   | REDIRECT_URI | OAuth 回调地址 | **需与管理员部署后的实际访问地址一致** |

3. **后台配置文件 `admin_config.json`**
   - 路径：由应用自动确定（项目目录或 `XQPS_CONFIG_DIR` 指定目录）
   - 内容：展示周期、公告、文档链接、模块开关等
   - **重要**：无论部署环境如何，管理员应通过系统内的「📋 后台配置」模块修改配置，请勿直接编辑该文件
   - 迁移时可将该文件拷贝给管理员作初始化参考，后续变更一律通过后台配置完成

4. **环境变量说明**
   - `APP_ENV`：生产用 `production`，测试用 `staging`
   - `ENABLE_DEMO_LOGIN`：生产建议 `false`，测试可 `true`
   - `REDIRECT_URI`：**必须与最终访问地址一致**（如 `http://管理员电脑IP:8501/` 或 `https://xqps.xxx.com/`）

### 2.2 飞书开放平台侧（如部署地址变更）

若迁移后**访问地址发生变化**（例如从你的电脑 IP 换成管理员电脑 IP 或新域名），需要在飞书开放平台修改：

- **应用管理** → **安全设置** → **重定向 URL**
- 添加或更新为新的访问地址（例如 `http://新IP:8501/` 或 `https://新域名/`）
- 确保与 `REDIRECT_URI` 完全一致

### 2.3 交接清单（可打印/截图给管理员）

- [ ] 代码来源：GitLab `prod` 分支 或 代码包
- [ ] 密钥清单（FEISHU_APP_ID、APP_SECRET、APP_TOKEN、TABLE_ID、REDIRECT_URI）
- [ ] `admin_config.json` 文件
- [ ] 当前运行方式说明（如：`streamlit run new_app.py --server.port 8501`）

## 三、接收方（管理员）需要做的工作

### 3.1 环境准备

1. **安装 Python 3.10+**
   - 确认：`python3 --version`

2. **克隆/解压代码**
   ```bash
   git clone http://git.snowballfinance.com/hr/xqps.git
   cd xqps
   git checkout prod
   ```
   或解压你提供的代码包到指定目录。

3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

### 3.2 配置密钥

在项目根目录创建 `.streamlit/secrets.toml`，或配置环境变量：

```toml
# .streamlit/secrets.toml 示例
FEISHU_APP_ID = "迁移方提供的值"
FEISHU_APP_SECRET = "迁移方提供的值"
FEISHU_APP_TOKEN = "迁移方提供的值"
FEISHU_TABLE_ID = "迁移方提供的值"
REDIRECT_URI = "http://本机IP或域名:8501/"
APP_ENV = "production"
ENABLE_DEMO_LOGIN = "false"
```

**重点**：`REDIRECT_URI` 必须填写**本机对外访问的完整地址**（含协议、IP/域名、端口）。

### 3.3 后台配置（通过系统内模块修改）

- **配置修改一律通过管理员后台「📋 后台配置」模块**完成，请勿直接编辑 `admin_config.json`
- 迁移时可将 `admin_config.json` 拷贝至项目根目录作初始化；若无该文件，首次运行后会自动生成空配置
- 管理员登录后，在侧边栏展开「📋 后台配置」即可修改展示周期、公告、文档链接、模块开关等

**部署环境（Docker/只读目录）**：若管理员无法保存公告、周期等配置，多为**目录无写权限**。请设置环境变量：
```bash
export XQPS_CONFIG_DIR=/data/xqps_config
```
并确保该目录可写、持久化（Docker 需挂载 volume）。启动时应用会在此目录生成 `admin_config.json`。管理员可在后台配置侧边栏查看当前配置路径并点击「🔧 测试写入」自检。

### 3.4 启动应用

```bash
streamlit run new_app.py --server.port 8501
```

或使用 nohup / systemd 等方式常驻运行（具体由管理员环境决定）。

### 3.5 接收方自检清单

- [ ] Python 3.10+ 已安装
- [ ] 依赖已安装（`pip install -r requirements.txt`）
- [ ] 密钥已配置（`.streamlit/secrets.toml` 或环境变量）
- [ ] `admin_config.json` 已就位
- [ ] `REDIRECT_URI` 与飞书应用后台「重定向 URL」一致
- [ ] 应用能正常启动并可访问
- [ ] 飞书登录授权正常（员工、管理员、HRBP 入口均可测试）

## 四、飞书侧需确认的配置（可选）

若迁移方无飞书应用管理权限，可请公司飞书管理员确认：

| 检查项 | 位置 | 说明 |
|--------|------|------|
| 应用可用性 | 飞书开放平台 → 应用 → 可用性 | 确保参与绩效的员工在可用范围内 |
| 重定向 URL | 应用 → 安全设置 → 重定向 URL | 必须包含新部署地址 |
| 权限 | 应用 → 权限管理 | 需有多维表格读取、用户信息等权限 |

## 五、迁移后验证

1. **员工登录**：用飞书账号点击「飞书授权登录」，能正常进入系统
2. **管理员登录**：点击右下角 👤 → 管理员登录，能进入后台
3. **HRBP 登录**：同上，HRBP 入口可进入
4. **数据与配置**：检查展示周期、公告、部门绩效等是否与迁移前一致

## 六、常见问题

| 问题 | 可能原因 | 处理 |
|------|----------|------|
| 授权失败 / 无法登录 | 重定向 URL 不匹配 | 检查 `REDIRECT_URI` 与飞书应用配置是否完全一致 |
| 管理员无法保存公告/配置 | 部署目录无写权限 | 设置 `XQPS_CONFIG_DIR` 指向可写目录（如 `/data/xqps_config`），Docker 需挂载 volume；所有配置应通过管理员登录后的「📋 后台配置」模块修改，勿直接编辑文件 |
| 当前考核周期暂无数据 | 展示周期与飞书数据不一致 | 在后台配置中点击「将 2026上半年 统一为 2026年上半年」或切换周期 |
| 提示无管理员权限 | 飞书表中「后台角色」未配置 | 在多维表中为对应人员填写「后台角色」字段 |
| 应用无法访问 | 防火墙 / 端口未开放 | 确保 8501 端口对访问用户开放 |
