# Tab 及页面内容梳理

> 基于 `new_app.py` 代码梳理。涵盖所有 Tab 的可见条件、页面结构及内容模块。

---

## 一、Tab 可见性总览

| 角色 | 可见 Tab |
|------|----------|
| **管理者** | 📝 员工自评、👥 上级评分、📌 一级部门负责人调整*、📌 分管高管调整*、📊 视图与报表、📂 团队历史绩效 |
| **员工** | 📝 员工自评、📂 团队历史绩效 |
| **员工个人** | 📝 员工自评 |

\* 一级部门负责人调整：仅 `is_dept_head` 时显示；分管高管调整：仅 `is_vp` 时显示

---

## 二、📝 员工自评

### 2.1 可见条件

- 所有角色可见（索引 0）

### 2.2 页面结构

```
🎯 当前绩效目标设定与自评
├── 已提交提示（自评是否提交=是时）
├── 💼 工作模块
│   ├── 工作目标1及总结、权重(%)、自评得分
│   ├── 工作目标2及总结、权重(%)、自评得分
│   ├── ... 工作目标5
│   ├── ➕ 添加工作目标（未提交时）
│   └── ➖ 删除最后目标（未提交时）
├── 🧠 能力模块
│   ├── 通用能力总结、通用能力自评得分
│   └── 领导力总结、领导力自评得分（仅管理者）
└── 操作区（未提交时）
    ├── 确认提交
    └── 保存草稿
```

### 2.3 内容说明

| 模块 | 说明 |
|------|------|
| 工作模块 | 3–5 个工作目标，每项：总结(100–5000字)、权重、自评得分；员工 80%、管理者 60% |
| 能力模块 | 通用能力 20%；管理者另有领导力 20% |
| 确认提交 | 权重+总结+得分合规后可提交，提交后表单锁定 |
| 保存草稿 | 暂存至飞书，不锁定 |

---

## 三、👥 上级评分

### 3.1 可见条件

- 仅管理者可见（`idx_mgr = 1`）

### 3.2 页面结构

```
👥 下属评估进展
├── 统计卡片：总下属、已交自评、已评、暂存、待评
├── 👇 下属评估名单
│   ├── 筛选：工号姓名、部门、状态、考核等级
│   ├── 确认提交（批量提交直属）
│   └── 列表：姓名(工号)、子部门/岗位、自评等级、考核等级、状态、操作
│       └── 操作：提醒 / 去评价 / 去调整 / 查看（直属/隔级不同）
└── 【展开】管理者评估下属的具体表单（选中下属时）
    ├── 👤 被评价人信息（工号、岗位、自评得分、自评等级、上一次绩效）
    ├── 💼 工作模块展示与评分
    ├── 🧠 通用能力模块展示与评分
    ├── 👑 领导力模块展示与评分（下属为管理者时）
    ├── 📈 考核总得分预览、绩效等级
    ├── ✍️ 考核评语
    └── 保存草稿
```

### 3.3 内容说明

| 模块 | 说明 |
|------|------|
| 下属评估进展 | 总下属、已交自评、已评、暂存、待评人数 |
| 下属名单 | 直属/隔级打标（隔级上级、一级部门负责人、分管高管）；隔级仅可查看 |
| 评估表单 | 工作/通用/领导力展示+上级评分；隔级下属只读 |
| 确认提交 | 仅直属，全部有等级且至少一人未提交时可点 |

## 四、📌 一级部门负责人调整

### 4.1 可见条件

- 仅 `is_dept_head` 时显示

### 4.2 页面结构

```
📌 一级部门负责人调整进展
├── 统计：覆盖人数、调整人数
├── 👇 待调整名单
│   ├── 筛选：工号姓名、部门、状态、考核等级
│   ├── 确认本次调整
│   └── 表格：姓名(工号)、部门/岗位、自评等级、考核等级、调整等级Ⅰ、操作
│       └── 操作：暂存、确认（行内）
└── 直属/隔级打标
```

### 4.3 内容说明

| 模块 | 说明 |
|------|------|
| 调整进展 | 覆盖人数（一级部门负责人名下）、调整人数（与上级评分不同的人数） |
| 待调整名单 | 一级部门负责人名下员工，可编辑一级部门调整考核结果 |
| 确认本次调整 | 全部待调整项有等级时可提交 |

---

## 五、📌 分管高管调整

### 5.1 可见条件

- 仅 `is_vp` 时显示

### 5.2 页面结构

```
📌 分管高管调整进展
├── 统计：覆盖人数、调整人数
├── 👇 待调整名单
│   ├── 筛选：工号姓名、部门（一级）、状态、考核等级
│   ├── 确认本次调整
│   └── 表格：姓名(工号)、一级部门/岗位、自评等级、考核等级、调整等级①、调整等级Ⅱ、操作
│       └── 操作：暂存、确认（行内）
└── 直属/隔级打标
```

### 5.3 内容说明

| 模块 | 说明 |
|------|------|
| 调整进展 | 覆盖人数（分管高管名下）、调整人数 |
| 待调整名单 | 分管高管名下员工；调整等级①=一级部门结果，调整等级Ⅱ=分管高管结果 |
| 确认本次调整 | 一级部门调整完毕=是 且 分管高管调整完毕≠是 的待调整项全部有等级时可提交 |

## 六、📊 视图与报表

### 6.1 可见条件

- 仅管理者可见

### 6.2 页面结构（普通管理者/隔级上级）

```
简化版
├── 🧭 绩效等级分布（柱状图）
├── 📈 考核进度统计
│   ├── 目标设定、自我评价、上级评价 完成数/总数
│   └── 进度条图
└── 🧾 部门绩效详情（表格：部门、总人数、已完成、完成率、S~C级）
```

### 6.3 页面结构（一级部门负责人/分管高管）

```
完整版
├── 📊 绩效概览
│   ├── KPI：考核总人数、已完成评价、总体完成率、剩余未评（可下钻）
│   └── 分管高管：销售/非销售筛选、分管范围总数配额表（S/A、B+、B+及以上、B、B-、C）
├── 分管高管：选择分管一级部门 → 各分管部门配额表
├── 团队成员
│   ├── 筛选：工号姓名、部门、状态、等级、每页、页码
│   └── 成员卡片（姓名、等级、部门、岗位、状态）
├── 📈 绩效视图
│   ├── 绩效等级分布（环形图）
│   └── 各部门考核完成率（条形图）
├── 🧾 部门绩效详情（可点击下钻）
└── 导出 CSV / Excel
```

### 6.4 内容说明

| 模块 | 说明 |
|------|------|
| 简化版 | 非一级部门负责人且非分管高管：等级分布、进度统计、部门详情 |
| 完整版 | 一级部门负责人或分管高管：KPI、配额表、成员卡片、图表、导出 |

## 七、📂 团队历史绩效

### 7.1 可见条件

- 管理者：可见
- 员工：可见（个人历史绩效）
- 员工个人：不可见

### 7.2 页面结构（管理者）

```
👇 下属历史绩效等级
├── 筛选：工号姓名、部门、考核等级、直属（隔级上级/一级部门负责人/分管高管有）
├── 列表：姓名(工号)、部门/岗位、考核周期、绩效等级
└── 直属打标（隔级上级/一级部门负责人/分管高管）
```

### 7.3 页面结构（员工）

```
个人历史绩效
├── 上一次考核周期、上一次绩效结果
└── ✍️ 上一次绩效考核评语
```

### 7.4 内容说明

| 角色 | 内容 |
|------|------|
| 管理者 | 下属历史绩效列表，可筛选、直属筛选；展示上一次绩效考核对应周期、结果 |
| 员工 | 本人上一次考核周期、结果、评语 |

---

