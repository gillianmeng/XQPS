# 管理者 - 权限与设置整理

> 用于整体权限调试阶段参考。基于 `new_app.py` 代码梳理。涵盖普通管理者、隔级上级、一级部门负责人、分管高管。

---

## 一、管理者类型与识别逻辑

### 1.1 识别来源

| 类型 | 识别条件 | 飞书字段 |
|------|----------|----------|
| **普通管理者** | 有直属下属，无隔级下属 | 直接评价人 |
| **隔级上级** | 有隔级下属（`my_all_subs` > `my_direct_subs`） | 隔级上级 |
| **一级部门负责人** | 任一员工的「一级部门负责人」包含当前用户 | 一级部门负责人 / 部门负责人 |
| **分管高管** | 任一员工的「分管高管」包含当前用户 | 分管高管 / 高管 |

### 1.2 数据范围

```python
# 直属：直接评价人 = 当前用户
# 隔级：隔级上级 = 当前用户 且 非直属
my_direct_subs   # 直属下级，可评价/调整
my_all_subs      # 直属 + 隔级，隔级仅展示
direct_record_ids = {s["record_id"] for s in my_direct_subs}
```

- **普通管理者**：`my_all_subs == my_direct_subs`，仅直属
- **隔级上级**：`len(my_all_subs) > len(my_direct_subs)`，含直属与隔级
- **一级部门负责人**：`is_dept_head = True`，按「一级部门负责人」字段
- **分管高管**：`is_vp = True`，按「分管高管」字段

## 三、上级评分（👥 上级评分）

### 3.1 下属范围

| 类型 | 可见下属 | 可评价/调整 | 可查看 |
|------|----------|-------------|--------|
| 普通管理者 | 直属 | 直属 | 直属 |
| 隔级上级 | 直属 + 隔级 | 直属 | 直属 + 隔级 |

### 3.2 操作权限

| 下属类型 | 提醒 | 去评价 | 去调整 | 查看 |
|----------|------|--------|--------|------|
| 直属 | ✅ | ✅ | ✅ | ✅ |
| 隔级 | ❌ | ❌ | ❌ | ✅ |

- **隔级下属**：仅可查看，表单只读（`disabled=not is_direct_sub`），侧边栏提示「隔级下属仅可查看，无调整权限」

### 3.3 直属/隔级打标

| 类型 | 是否打标 | 说明 |
|------|----------|------|
| 普通管理者 | ❌ | 仅直属，无需区分 |
| 隔级上级 | ✅ | 直属绿色(直属)、隔级灰色(隔级) |
| 一级部门负责人 | ✅ | 同上 |
| 分管高管 | ✅ | 同上 |

### 3.4 确认提交

- **范围**：仅 `my_direct_subs`（直属）
- **条件**：全部直属有考核等级 且 至少一人未提交
- **全部已提交**：按钮置灰，不可再提交

## 四、一级部门负责人调整（📌 一级部门负责人调整）

### 4.1 可见条件

- `is_dept_head = True`
- 否则提示：「当前您不是任何员工的一级部门负责人，暂无可调整名单。」

### 4.2 调整范围

- **纳入条件**：员工「一级部门负责人」包含当前用户，且非本人
- **部门口径**：按二级部门（二级-三级-四级链路）展示

### 4.3 操作权限

- 可修改「一级部门调整考核结果」
- 可确认「一级部门调整完毕」= 是
- 默认取上级评分结果，可编辑覆盖

### 4.4 直属/隔级打标

- 直属：绿色 (直属)
- 隔级：灰色 (隔级)
- 判断：`直接评价人` = 直属，`隔级上级` = 隔级

---

## 五、分管高管调整（📌 分管高管调整）

### 5.1 可见条件

- `is_vp = True`
- 否则不显示该 Tab

### 5.2 调整范围

- **纳入条件**：员工「分管高管」包含当前用户，且非本人
- **部门口径**：按一级部门展示
- **排除**：不排除其他分管高管（可见名下全部员工）

### 5.3 操作权限

- 可修改「分管高管调整考核结果」
- 可确认「分管高管调整完毕」= 是
- 默认取「一级部门调整考核结果」，可编辑覆盖

### 5.4 直属/隔级打标

- 同一级部门负责人

## 七、团队历史绩效（📂 团队历史绩效）

### 7.1 下属范围

| 类型 | 纳入条件 |
|------|----------|
| 分管高管 | 「分管高管」包含当前用户，排除其他分管高管本人 |
| 一级部门负责人 | 「一级部门负责人」包含当前用户 |
| 普通管理者/隔级上级 | 「直接评价人」或「隔级上级」包含当前用户 |

### 7.2 直属筛选与打标

| 类型 | 直属按钮 | 直属打标 |
|------|----------|----------|
| 普通管理者 | ❌ | ❌ |
| 隔级上级 | ✅ | ✅ |
| 一级部门负责人 | ✅ | ✅ |
| 分管高管 | ✅ | ✅ |

- **直属筛选**：`has_skip_level_subs = len(my_all_subs) > len(my_direct_subs)` 或 `is_vp` 或 `is_dept_head`
- **直属打标**：同上，仅直属显示绿色 (直属)

### 7.3 展示内容

- 下属历史绩效等级列表
- 筛选：工号姓名、部门、考核等级
- 直属模式下仅显示直属下属

## 八、员工自评（管理者视角）

### 8.1 自评表单

- 工作模块 60%、通用能力 20%、领导力 20%
- 与员工（80%+20%）权重不同

### 8.2 验证模块

- 评估直属时：显示下属验证（工作权重、能力、领导力等）
- 评估隔级时：显示「隔级下属仅可查看，无调整权限」
