# XQPS 交接清单：维护人张燕职责与待办

本文档与 **[交接与使用指南.md](./交接与使用指南.md)**、**[README.md](../README.md)** 配合使用。  
**技术细节、环境变量表、风险说明**以《交接与使用指南》为准；本文侧重 **谁做什么、对方必须完成的条目**。

---

## 一、角色约定

| 角色 | 建议职责 |
|------|-----------|
| **主维护人张燕** | 日常需求、排障、发版、飞书应用与多维表对接、密钥保管与轮换的第一责任人 |
| **原负责人孟凡卓** | 移交代码、过渡期答疑 |
| **业务方=主维护人** | 表结构变更、制度与周期口径的产品决策 |

---

## 二、主维护人张燕必须完成的事项

可逐项打勾并记录完成日期与账号名。

### A. 代码与仓库

- [ ] 本地能 **安装依赖并成功运行**：`pip install -r requirements.txt` 后 `streamlit run new_app.py`（端口与 `REDIRECT_URI` 一致，见下文）。
- [ ] 阅读并遵守 **勿提交**：`.streamlit/secrets.toml`、真实 `demo_users*.json`、任何密钥文件（见 `.gitignore` 与《交接与使用指南》）。
- [ ] 获得 **GitLab（`snowball`）** 的 **Developer** 权限，能 `clone`、`push` 到约定分支（当前常用 **`prod`**）。

### B. 飞书开放平台（应用负责人应为主维护人张燕）

- [ ] 能登录 **[飞书开放平台](https://open.feishu.cn/)** 并打开 **本系统使用的自建应用**。
- [ ] 确认自己为 **应用管理员** 或具备 **修改应用凭证、权限、版本发布** 的权限；若不能，协调 **原负责人或企业管理员** 将应用 **转移负责人** 或 **加你为管理员**（以贵司规范为准）。
- [ ] **应用凭证**：保存 **App ID、App Secret** 至公司认可的 **密码管理器或密钥库**（禁止长期明文放在聊天或未加密文档）。
- [ ] **重定向 URL（Redirect URI）**：与部署地址一致。本地开发示例：`http://localhost:8501/`（**须与代码中 `REDIRECT_URI` 完全一致**，含末尾斜杠以实际配置为准）。
- [ ] **权限范围（scope）**：包含本应用所需的 **多维表读写**、**用户身份/OAuth** 等（以当前线上已开通的为准，变更需测试登录与拉表）。
- [ ] 应用有 **版本发布** 流程的，权限变更后按平台要求 **创建版本并发布**。
- [ ] 约定 **Secret 轮换** 周期与流程（例如每 12 个月或泄露后立即轮换），并记录 **谁执行、谁更新部署环境**。

### C. 飞书多维表（Bitable）

- [ ] 明确 **数据归属**：Base 在 **哪个企业**、**谁有管理员权限**。
- [ ] 记录并保管 **App Token、Table ID**（与 `FEISHU_APP_TOKEN`、`FEISHU_TABLE_ID` 对应）；**表迁移或复制后这两个通常会变**，必须同步改环境变量并回归测试。
- [ ] 若发生 **字段增删改、选项变更**：先知会开发评估对 `new_app.py` 解析逻辑的影响（字段名与业务规则见 **[飞书表头与模块对应关系.md](./飞书表头与模块对应关系.md)** 等文档）。
- [ ] 为应用开通 **对该 Base/表的访问授权**（机器人/应用身份可读写），与开放平台文档一致。

### D. 运行环境与配置（主维护人张燕）

- [ ] 在 **测试环境** 与 **生产环境** 分别配置完整环境变量或 `secrets.toml`（必填项见《交接与使用指南》第 3 节）。
- [ ] 确认 **`REDIRECT_URI`** 与开放平台、实际访问域名 **三者一致**。
- [ ] 生产环境：**`APP_ENV=production`**，**`ENABLE_DEMO_LOGIN=false`**（避免演示入口暴露）。
- [ ] 若使用 **Docker/只读盘**：配置 **`XQPS_CONFIG_DIR`** 指向 **可写挂载目录**，否则后台配置无法保存（见《交接与使用指南》）。
- [ ] 保存一份 **脱敏** 的环境变量说明（不写 Secret 真值，只写变量名与用途），随仓库或内部 wiki 更新。

### E. 部署与发版（主维护人张燕）

- [ ] 掌握 **当前生产** 的启动方式（脚本、systemd、K8s 等）与 **日志位置**。
- [ ] 约定 **发版流程**：从 `prod` 拉取、构建、重启、验证登录与关键路径。
- [ ] 约定 **回滚**：上一版本镜像/分支如何恢复。

### F. 业务与协作

- [ ] **业务方=主维护人** 接手系统权限及开发。
- [ ] 保留 **内部联系人**：飞书企业管理员（魏志伟）、网络/域名负责人HTTPS 证书、防火墙（牛翔山）。

## 三、原负责人孟凡卓交接期配合的事项

- [ ] 移交 **开放平台应用负责人**。
- [ ] 移交 **[GitLab](http://git.snowballfinance.com/hr/xqps)**、**[域名/DNS](http://rc-hrps.snowballfinance.com/)** 、**[雪橇部署平台](https://luge.snowballfinance.com/project/xueqiu-performance-system)** 的权限或工单。（牛翔山）
- [ ] 提供 **当前生产环境变量键名列表**（Secret 通过安全渠道单独交接，不贴仓库）。
- [ ] 说明 **历史债务**：已知 bug、临时开关、仅生产存在的配置，**[绩效系统测试汇总](https://xueqiu.feishu.cn/base/OlRnbwA0PaydVysrS7WcCwP6nOc?table=tblnYEPXpjXWrR6l&view=vew1RmaV9G)**。
- [ ] 交接后 **保留顾问期**（2周）

## 四、主维护人张燕「必须准备」的材料

1. 可用于接收权限的 **飞书 / 开放平台账号**（与企业主体一致）。
2. 是否接受担任 **应用负责人** 或等价职责（见第二节 B）。
3. 本机或公司开发机是否允许安装 **Python 3.10+** 与访问 **飞书 API**（含代理策略）。
4. 生产部署的 **URL** 与 **HTTPS** 是否已就绪（与 `REDIRECT_URI` 绑定）。

## 五、验收标准（交接完成判定）

建议双方签字或邮件确认：

- [ ] 主维护人 **独立** 完成一次 **从仓库拉代码到本地运行**（测试环境 OAuth 成功）。
- [ ] 主维护人 **独立** 完成一次 **生产或预发配置核对**（不泄露 Secret 的前提下核对变量键与回调 URL）。
- [ ] **密钥与表权限** 的责任人书面为主维护人（或公司指定的运维角色）。
